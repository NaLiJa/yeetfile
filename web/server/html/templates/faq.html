FAQ

How does YeetFile work?

When you upload a file using YeetFile, it generates a key using a randomly generated passphrase and (if password protection is enabled) your password. Your file is then split into individual “chunks”, and each chunk is encrypted before being sent to the server. The server passes along these encrypted chunks of data to Backblaze for storage. Once either the download limit or expiration date has been reached, the file is deleted from Backblaze, and all metadata pertaining to the file is deleted from the server’s database.

What file metadata is stored in YeetFile’s database?

YeetFile stores:

  • The number of chunks that the file was split up into
  • The total file size, in bytes
  • The encrypted filename

The filename is encrypted with the same key that file data is encrypted with, and since the server never gains access to the randomly generated passphrase or (if provided) the password, it is unable to ever decrypt the filename.

What account metadata is stored in YeetFile’s database?

YeetFile stores:

  • The account number
  • The remaining monthly storage for the account
  • The account type (aka which tier)
  • The subscription expiration date (if applicable)
  • The email address (if supplied)
  • The password hash (if supplied)
Are files that I upload tied to my account specifically?

No, file uploads are not tied to your account. The database does not associate account numbers with files when they’re uploaded. Account usage is determined by your uploaded file sizes, but these values are rounded down to the nearest megabyte and thus do not map perfectly to the actual uploaded file size.

Should I use the website or the command line application? What’s the difference?

It’s up to you which one you use, but the command line application has the added security of not requiring external resources to be downloaded in order for it to work. Using the website requires trusting that the JavaScript files sent from the server each time you access the site are the same each time, and that nothing has been modified to alter the site’s behavior.

As far as differences go, there are none. All features supported in the website are also supported in the command line application.

Should I sign up with an email/password or just request an account number?

Whatever works best for you. With an email/password, you can always recover your account if you lose access to your password. Signing up with only an account number means if you lose your account number, you lose access to your account permanently.

Note that you can always add an email and/or password to your account later if you signed up with only an account number at first.

What emails are sent by YeetFile?

The only emails that YeetFile can send are the verification email and the password reset email.

Do upgrades to my meter expire?

No, any upgrades you purchase are free to use until your meter runs out.

Is YeetFile open source?

Yes, the source for YeetFile is AGPL licensed and available on GitHub and SourceHut.

Why does YeetFile charge for usage if it’s open source? Can I run my own instance for free?

Although the code is free and open source, there are plenty of operating costs (server fees, storage fees, data transfer fees, etc.) tied to running a reliable public instance.

If you want to run an instance of YeetFile yourself, there are instructions in the README for deploying your own instance, with configuration options such as max upload size, where to store uploads, and more.

Why metered usage? Why not a monthly or yearly subscription?

A subscription model would be unfair to the user, since a user is essentially paying for storage and egress (which are both costs based on usage). Metered usage allows users to use YeetFile however much they need, without worrying about using it within a specific amount of time.

Can I pay in crypto?

Yes, there's a BTCPay server set up for YeetFile that can accept Bitcoin and Monero payments.

Why should I trust YeetFile with my files?

YeetFile is designed to not really require any trust, since the server cannot decrypt file content.

That being said, if you truly want to make sure that the system is working as it should (encrypting files locally before upload, not sending key data anywhere else, etc.), you can verify and build the source code for the command line application yourself and use it to upload/download files.

Who made YeetFile?

YeetFile was developed by Ben Busby, the developer of other open source software such as Whoogle Search and Farside, and more.

YeetFile was released under Ben Busby's open source software collective, meddlehead.
